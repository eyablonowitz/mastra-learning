# Workspace plans

Keep implementation plans for this workspace in this directory using the naming convention:

```text
YYYY-MM-DD-HHMMSS-descriptive-plan-name.md
```

Timestamps use the local workspace timezone (`America/New_York`) and 24-hour time. Plan names should be lowercase, concise, and hyphen-separated.

Each plan should capture:

- The intended outcome and scope
- Important architecture or implementation decisions
- An ordered implementation plan
- Verification criteria
- Explicitly deferred work

Plans are historical decision records. Prefer adding a new timestamped plan when direction changes substantially; update an existing plan only while refining the same piece of work.
